---
description: Puppeteerでヘッドレスブラウザ自動化・E2Eテスト
---

**Puppeteer MCPサーバ**を使用して、ヘッドレスブラウザでのE2Eテストと自動化を実行します。

## 使用方法
このコマンドの後に、実行したいE2Eテストシナリオを指定してください。

## 主な機能

### 1. E2Eテスト自動化
- ユーザーフロー全体のテスト
- マルチステップシナリオ
- 異常系テスト

### 2. パフォーマンステスト
- ページロード時間測定
- APIレスポンスタイム
- リソース使用量監視

### 3. レポート生成
- スクリーンショット自動撮影
- PDF レポート生成
- テスト結果サマリ

### 4. CI/CD統合
- 自動テストパイプライン
- 回帰テスト
- スモークテスト

## 主なテストシナリオ

### E2E-001: 管理GUI完全フロー
1. ログイン
2. PC一覧表示
3. CSV インポート
4. データ確認
5. ODJファイルアップロード
6. ログアウト

### E2E-002: API統合テスト
1. /api/pcinfo 呼び出し
2. レスポンス検証
3. /api/log でログ記録
4. データベース確認

### E2E-003: エラーハンドリング
1. 不正なSerial番号で404確認
2. 必須パラメータなしで400確認
3. エラーメッセージ表示確認

## 例

### 管理GUI E2Eテスト
```
/mcp-puppeteer
管理GUIの全機能（ログイン、CSV インポート、PC登録、検索、ODJアップロード）をE2Eテストしてください
```

### パフォーマンス測定
```
/mcp-puppeteer
管理GUIの各ページのロード時間を測定し、200ms以内であることを確認してください
```

### 回帰テスト
```
/mcp-puppeteer
すべてのE2Eテストシナリオを実行し、前回のテスト結果と比較してください
```

### スクリーンショットレポート
```
/mcp-puppeteer
管理GUIの全画面のスクリーンショットを撮影し、PDFレポートを生成してください
```

Puppeteerで完全自動化されたE2Eテストを実現します。
