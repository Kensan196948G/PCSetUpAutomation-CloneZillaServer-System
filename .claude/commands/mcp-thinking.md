description: Sequential Thinkingで複雑な問題を段階的に解決
---

**Sequential Thinking MCPサーバ**を使用して、複雑な技術問題を段階的に分析・解決します。

## 使用方法
このコマンドの後に、解決したい複雑な問題や設計課題を指定してください。

## 主な用途

### 1. アーキテクチャ設計
- システム全体の設計判断
- コンポーネント間の関係性分析
- スケーラビリティ検討

### 2. パフォーマンス最適化
- ボトルネック特定
- 最適化戦略立案
- トレードオフ分析

### 3. セキュリティ分析
- 脅威モデリング
- 脆弱性分析
- 対策優先順位付け

### 4. トラブルシューティング
- 根本原因分析
- 複数要因の関連性分析
- 解決策の段階的導出

### 5. 技術選定
- 技術スタック比較
- プロ・コン分析
- 長期的影響評価

## 適用場面

### システムアーキテクチャ
- PXEブート〜ドメイン参加の最適フロー設計
- API設計の一貫性確保
- エラーハンドリング戦略

### パフォーマンス
- 10台同時展開時のボトルネック分析
- API応答時間200ms達成戦略
- データベースクエリ最適化

### セキュリティ
- ODJファイルアクセス制御設計
- API認証方式選定
- ログ記録戦略

## 例

### アーキテクチャ最適化
```
/mcp-thinking
PXEブートからドメイン参加完了までの全フローを分析し、各ステップの所要時間を最小化する最適化戦略を段階的に導出してください
```

### パフォーマンスボトルネック分析
```
/mcp-thinking
10台同時展開時に性能劣化が20%を超える原因を段階的に分析し、ボトルネックを特定してください。ネットワーク、ディスクI/O、CPU、メモリの各観点から検討してください
```

### セキュリティリスク分析
```
/mcp-thinking
社内LAN限定のAPI設計について、セキュリティリスクを段階的に分析し、追加すべき対策を優先順位付けしてください
```

### 技術選定
```
/mcp-thinking
データベースとしてSQLite vs PostgreSQLを比較し、このプロジェクトの要件（年間100台、単一サーバ、シンプル運用）に最適な選択を段階的に導出してください
```

### エラーハンドリング戦略
```
/mcp-thinking
API不応答、ODJ適用失敗、ネットワーク切断の3つのエラーケースについて、最適なハンドリング戦略を段階的に設計してください。リトライ、ログ記録、ユーザー通知の各側面から検討してください
```

### 複雑なバグ分析
```
/mcp-thinking
Sysprep実行後にドメイン参加が失敗する問題について、考えられる原因を段階的に列挙し、検証手順と解決策を導出してください
```

Sequential Thinkingで複雑な問題を体系的に解決します。
