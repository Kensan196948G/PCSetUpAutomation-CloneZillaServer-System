description: Memoryでプロジェクト知識を永続的に記録・管理
---

**Memory MCPサーバ**を使用して、重要なプロジェクト情報を永続的に記録・管理します。

## 使用方法
このコマンドの後に、記録・取得したい情報を指定してください。

## 主な機能

### 1. 設計判断の記録
- アーキテクチャ選択理由
- 技術スタック選定理由
- パフォーマンス最適化の根拠
- セキュリティ対策の実装内容

### 2. トラブルシューティング履歴
- 過去の問題と解決策
- バグ修正履歴
- パフォーマンス改善履歴

### 3. API設計変更履歴
- エンドポイント変更理由
- レスポンス形式変更
- バージョニング戦略

### 4. 運用ノウハウ
- デプロイ手順
- バックアップ戦略
- 監視・アラート設定

## 記録すべき重要情報

### アーキテクチャ関連
- SQLiteを選択した理由
- Flask vs FastAPI の検討結果
- ODJファイルの保存場所決定理由

### パフォーマンス関連
- API応答時間200ms以内の達成方法
- 10台同時展開の性能チューニング
- データベースインデックス設計

### セキュリティ関連
- API認証方式の選択理由
- ODJファイルアクセス制御
- 社内LAN限定の実装

## 例

### 設計判断を記録
```
/mcp-memory
「SQLiteを選択した理由: 小規模（年間100台）、シンプル運用、バックアップ容易」を記録してください
```

### トラブルシューティング記録
```
/mcp-memory
「Sysprep失敗の解決策: AppXパッケージ事前削除、プロビジョニングパッケージ削除」を記録してください
```

### API変更履歴記録
```
/mcp-memory
「/api/pcinfoにodj_path追加（2025-11-17）: ODJファイル動的取得のため」を記録してください
```

### パフォーマンス改善記録
```
/mcp-memory
「Serial番号にINDEX追加で検索速度50ms→5msに改善」を記録してください
```

### 記録内容の取得
```
/mcp-memory
これまでに記録されたパフォーマンス最適化の履歴をすべて取得してください
```

Memoryで重要な知識を永続化し、将来の開発に活かします。
