description: 全テスト実行（ユニット、統合、E2E）
---

すべてのテストを実行して、システムの品質を確認します。

**test-engineer** および **integration-tester** エージェントを起動して、以下を実行してください：

## 1. ユニットテスト
- Pythonテスト（pytest）
  - Flask API テスト
  - データベース操作テスト
  - バリデーションテスト
- PowerShellテスト（Pester）
  - 各関数のユニットテスト
  - モックを使用したAPIテスト

## 2. 統合テスト
- API統合テスト
  - GET /api/pcinfo エンドポイント
  - POST /api/log エンドポイント
  - エラーレスポンステスト
- データベース統合テスト
  - CRUD操作
  - トランザクション

## 3. E2Eテスト
- 完全自動キッティングフロー
  1. PXEブート（シミュレーション）
  2. イメージ展開
  3. Windows初回起動
  4. Serial取得 → API呼び出し
  5. PC名設定
  6. ODJ適用
  7. 完了ログ送信
  8. 検証

## 4. パフォーマンステスト
- API応答時間（目標: 200ms以下）
- 同時展開（10台）のシミュレーション

## 5. レポート作成
- テストカバレッジレポート
- 失敗テスト一覧
- パフォーマンス測定結果

すべてのテスト結果をサマリとして報告してください。
