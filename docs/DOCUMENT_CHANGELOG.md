## 変更履歴の使い方

### 本ドキュメントの目的

このドキュメントは、会社キッティング自動化フレームワークプロジェクトのすべてのドキュメント変更を一元管理します。

### 更新タイミング

**必須更新**: 以下の場合、このCHANGELOGを更新してください
- ドキュメントの新規作成
- ドキュメントの内容変更（軽微な誤字修正を除く）
- ドキュメントの削除・移動
- ドキュメント構造の変更

**推奨更新**: 以下の場合も記録推奨
- 誤字・脱字の修正（複数箇所）
- レイアウトの変更
- リンクの追加・修正

### 記録方法

1. 該当するセクション（[カテゴリ別変更履歴](#カテゴリ別変更履歴)）を見つける
2. [変更記録テンプレート](#変更記録テンプレート)をコピー
3. 必要事項を記入
4. 最新の変更が一番上になるように追記

## 変更管理ルール

### バージョン番号体系

本プロジェクトでは、**セマンティックバージョニング**を採用します。

```
X.Y.Z
│ │ └─ パッチバージョン（バグ修正、誤字修正）
│ └─── マイナーバージョン（機能追加、後方互換性あり）
└───── メジャーバージョン（大幅変更、後方互換性なし）
```

### バージョンアップの判断基準

#### パッチバージョン（v1.0 → v1.0.1）
**変更内容**:
- 誤字・脱字の修正
- リンク切れ修正
- 軽微な表現の改善
- コマンド例の誤り修正

**承認者**: ドキュメント作成者本人
**通知**: 不要（月次レポートで報告）

---

#### マイナーバージョン（v1.0 → v1.1）
**変更内容**:
- 新しいセクションの追加
- 手順の詳細化
- 新しいコマンド例の追加
- 図表の追加
- 既存機能の改善（後方互換性維持）

**承認者**: カテゴリ責任者（開発リーダー、運用責任者等）
**通知**: Slack通知 + 月次レポート

---

#### メジャーバージョン（v1.x → v2.0）
**変更内容**:
- ドキュメント構造の大幅変更
- アーキテクチャの全面見直し
- 非互換な仕様変更
- プロセスの根本的変更

**承認者**: 技術責任者、プロジェクトマネージャー
**通知**: メール + ミーティング + 月次レポート

### 変更時の必須作業

1. **変更前**:
   - [ ] 該当ドキュメントのバックアップ作成
   - [ ] 変更理由の明確化
   - [ ] 影響範囲の特定

2. **変更中**:
   - [ ] ドキュメント内容の更新
   - [ ] バージョン番号の更新
   - [ ] 最終更新日の更新

3. **変更後**:
   - [ ] 本CHANGELOGへの記録
   - [ ] 関連ドキュメントのリンク確認
   - [ ] 承認者へのレビュー依頼
   - [ ] 関係者への通知
