キッティング業務フロー図（BPMN風）

以下は、「PC納品」から「ユーザー渡し」までの業務フローを、BPMNライクに整理したもの。

スイムレーン表現

レーン 担当 主なタスク
総務/購買 総務部
  • PCの発注
  • 納品物の受け取り・検品
  • PC箱の保管・ITシステム部門への引き渡し
ITシステム部門（前処理） IT部門
  • 箱側面のSerial番号をスマホで撮影
  • ChatGPTでSerial OCR → CSV化
  • 導入日ベースPC名生成（YYYYMMDDM）
  • 管理GUIにCSVインポート
管理システム（DRBL/GUI） システム
  • pc_masterテーブルに Serial/PC名/ODJパス登録
  • ODJファイルの格納
  • API /pcinfo /log 提供
DRBL/Clonezillaサーバ システム
  • PXEブート提供
  • マスターイメージの配信
  • 展開ログの保持（オプション）
WindowsクライアントPC PC本体
  • PXEブートでClonezilla起動
  • マスターイメージ受信
  • 初回起動 → Serial取得・API問い合わせ
  • PC名設定・ODJ適用・再起動
  • Windows Update・アプリ導入
ITシステム部門（確認） IT部門
  • 管理GUIでステータス確認
  • 不良があれば再展開 or 再実行
  • ユーザーへPC引き渡し

簡易BPMN（テキスト表現）

開始イベント: PC納品
  ↓
総務：PC検品・IT部門へ引き渡し
  ↓
IT部門：箱Serial撮影 → ChatGPT OCR → PC名生成(YYYYMMDDM) → CSV作成
  ↓
IT部門：管理GUIへCSVインポート（pc_master登録）
  ↓
AD管理者：PC名ごとのODJファイル生成 → DRBLサーバへ配置
  ↓
IT部門：PCを展開用LANへ接続しPXEブート
  ↓
DRBL/Clonezilla：マスターイメージ展開
  ↓
Windows PC：初回起動 → 初期スクリプト実行
  ↓
スクリプト：Serial取得 → API /pcinfo → PC名決定＆ODJパス取得
  ↓
スクリプト：PC名設定 → ODJ適用 → 再起動
  ↓
Windows PC：ドメイン参加済みで起動
  ↓
スクリプト：Windows Update / アプリ導入 / 完了ログ送信
  ↓
IT部門：管理GUIでステータス確認
  ↓
ユーザーへ引き渡し
  ↓
終了イベント：キッティング完了
